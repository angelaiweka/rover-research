Reinforcement learning for autonomous planetary rover navigation in simulated unstructured terrain

In progress ROS2 · Gazebo · PPO · Stable-Baselines3 Arizona State University

Can a reinforcement learning agent learn a navigation policy that generalizes to terrain it has never seen? This project investigates onboard autonomous decision-making for planetary rovers operating under deep-space communication constraints.

Overview
Research question

How can reinforcement learning be used to train a simulated planetary rover to navigate autonomously through unstructured terrain, without relying on real-time input from a human operator?

Hypothesis

A PPO agent trained on randomized Gazebo terrain will achieve a higher goal-reaching success rate and lower collision frequency than a rule-based baseline, across held-out terrain configurations.

Motivation

Planetary rovers operating on Mars cannot rely on real-time communication with Earth. One-way signal travel times range from 4 to 24 minutes, making teleoperation impractical in time-sensitive situations. Current missions address this by uploading scripted command sequences in advance, but these approaches break down when terrain conditions do not match what was anticipated.

This project investigates reinforcement learning as an approach to onboard autonomy: rather than being explicitly programmed for every scenario, the rover learns navigation behaviors through trial and error in simulation. The central challenge is generalization: ensuring the learned policy performs reliably on environments it was not trained on.

Methods
Tools and environment

Fully computational, no hardware or participants required. All experiments run in simulation on a personal laptop (Ubuntu 24 via UTM on macOS). ASU Research Computing may be used for extended GPU training.

Python ROS2 Humble Gazebo Stable-Baselines3 PyTorch TurtleBot3
Procedure
Wks 1–2

Environment setup

Install and configure ROS2 Humble and Gazebo. Load a TurtleBot3 rover model and build a simulated terrain environment with randomized obstacle placement and surface irregularities.

Wks 3–4

Baseline controller

Implement a reactive rule-based controller using laser scan data. Run 50 randomized terrain trials and record goal-reaching success rate, path length, and collision count.

Wks 5–8

RL agent development

Define the observation space (laser scan ranges, relative goal position, current velocity), action space (linear and angular velocity commands), and reward function. Train a PPO agent across randomized terrain for at least 500,000 timesteps with checkpoint logging.

Wks 9–10

Evaluation

Test the trained policy on 50 held-out terrain configurations not seen during training. Compare goal-reaching success rate, path efficiency, and collision frequency against the rule-based baseline. Analyze failure cases.

Wks 11–14

Analysis and documentation

Produce training curve and behavior visualizations. Publish the full pipeline to GitHub and prepare the final report.
